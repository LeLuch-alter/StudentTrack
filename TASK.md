# Описание проекта
Необходимо разработать консольное приложение для управления базой данных студентов, их курсов и оценок. Все данные хранятся в памяти приложения во время его работы — используйте коллекции List и Map. Внешние базы данных не требуются.

# Архитектурные требования
Проект разбит на 4 таска. Реализуйте их по порядку.

# Таск 1 — Доменная модель (Сущности)
Создайте три класса:
-	Student — поля: id (int), name (String), group (String)
-	Course — поля: courseCode (String, напр. INF-101), courseName (String)
-	Grade — связывает студента, курс и оценку (int, от 0 до 100)
Требования: все поля private, доступ через геттеры/сеттеры, метод toString() обязателен в каждом классе.

# Таск 2 — Бизнес-логика (Сервисный слой)
Создайте класс TrackerService. В нём реализуйте следующие методы:
-	addStudent(Student student) — добавление нового студента
-	addCourse(Course course) — добавление нового предмета
-	assignGrade(int studentId, String courseCode, int score) — выставить оценку
-	getStudentGpa(int studentId) — расчёт среднего балла студента

# Таск 3 — Аналитика и поиск
Добавьте в TrackerService аналитические методы:
-	Список всех студентов, у которых GPA выше заданного порога
-	Топ-3 студентов с наивысшей успеваемостью
-	Список предметов и средний балл по каждому из них

# Таск 4 — Консольный интерфейс
В классе Main реализуйте меню через Scanner и цикл while(true):
1.	Добавить студента
2.	Добавить предмет
3.	Выставить оценку
4.	Показать средний балл студента
5.	Показать топ-3 студентов
6.	Аналитика по предметам
7.	Выход

# Технические рекомендации
-	ArrayList<Student> и ArrayList<Course> — для хранения данных
-	HashMap<Integer, Student> — для быстрого поиска по ID
-	Оценка строго от 0 до 100, иначе выводить ошибку
-	Если studentId не существует — выводить сообщение, не падать
-	Ввод чисел оборачивать в try-catch (NumberFormatException)
-	Имена переменных и методов только на английском: studentName, calculateGpa()
-	Структура пакетов: model, service, main

# Критерии оценивания

Критерий	Описание	Вес
Принципы ООП	Инкапсуляция, разделение на слои (model / service / main)	25%

Валидация данных	Программа не падает при неверном вводе	25%
